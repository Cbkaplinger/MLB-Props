# Strikeout-EDA

Exploratory analysis for the pregame pitcher strikeout-rate model.

## Folder naming convention

Each modeled statistic gets its own `<Statistic>-EDA` folder under its model
directory, e.g.:

- `Models/Strikeout-Model/Strikeout-EDA/`  (this one)
- `Models/Walk-Model/Walk-EDA/`
- `Models/Hit-Model/Hit-EDA/`
- `Models/Out-Model/Out-EDA/`

Keep this convention for walks, hits, and outs as those models are built.

## Contents

- `strikeout-eda.ipynb` - target distribution, correlations, skew/null audits,
  physics-vs-K scatter, stat stabilization curves, and a per-stat reliability
  table (split-half, ICC, year-over-year, SEM, regression slope, feature tiers).
- `Graphs/` - figures regenerated by running the notebook (written next to the
  notebook via `__vsc_ipynb_file__`, so renaming the folder never breaks paths).

## Running

From the repo root, with the project venv
(`pip install -e ".[research,dev]"`), open the
notebook and select the project environment. It reads the canonical Level 1
artifact at `mlb_props.config.PITCHER_GAMES_PATH`. Run
`python -m mlb_props.pipeline.games` first.

## Where the logic lives

Reusable reliability/stabilization functions were extracted to
`src/mlb_props/reliability.py` (unit-tested in `tests/test_reliability.py`) so
feature-selection code can call the same routines the EDA uses:

- `stabilization_curve(...)` - split-half reliability vs window size, all stats at once.
- `reliability_table(...)` - split-half + Spearman + YoY + SEM + slope + tier per stat.
- `stabilization_by_denominator(...)` - pitch/zone/swing/PA-denominated curves
  used to choose candidate rolling windows.

> **Data note:** Training window is the 2023-2025 regular seasons; the target
> deployment is 2026 projections (see `mlb_props.config.TRAIN_SEASONS` /
> `PROJECTION_SEASON`).
>
> **Leakage note:** `PA`, `K`, and other same-game outcomes appear here only for
> analysis. They must never become pregame model features.
